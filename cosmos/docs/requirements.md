# 宇宙科普站点需求

## 背景

在当前项目中单独创建一个宇宙科普 Web 子站点，作为视觉质量更高的 Product Design 原型。站点不替换已有小学数学站点，独立位于 `cosmos/`。

## 目标

- 第一屏呈现惊艳的宇宙科普体验。
- 使用方案 2「Cosmic Story Atlas / 星图课堂」作为视觉目标。
- 内容重点包括宇宙尺度、恒星一生、黑洞边界、星系地图、探索任务。
- 首屏必须具备可交互元素，而不是静态海报。

## 第一版范围

- React + Vite 单页站点。
- 沉浸式全屏星云/星系视觉。
- 顶部导航、搜索入口、用户入口。
- 左侧主标题和 CTA。
- 中央宇宙尺度时间轴，可切换尺度节点。
- 右侧课程面板，可切换课程。
- 底部探索任务轨，可切换任务状态。
- 数据面板随当前节点联动。

## 非目标

- 不做后端、账号系统、真实课程播放。
- 不引入重型 3D 引擎。
- 不覆盖根目录数学站点。

## 范围演进（约束放宽）

- 第二阶段起站点定位为「可持续探索的宇宙知识站」，强调严谨且引人停留的内容。
- 已从单页扩展为多路由（首页 / 专题详情 / 冷知识流），原「不做多页面路由」约束取消。
- 内容以 `src/data/topics.js` 为单一真相来源，覆盖常规宇宙知识与科学严谨的冷知识。

## 验收标准

- `npm run build` 通过。
- `cosmos/` 内有完整需求、设计、开发文档。
- 页面在桌面首屏具备深空视觉冲击力。
- 至少 3 类交互可用：尺度节点、课程卡、任务轨。
- 移动端不出现主要文字重叠。
