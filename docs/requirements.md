# 小学数学可视化站点需求

## 背景

面向小学 1-5 年级学生，构建一个以人教版小学数学知识脉络为参考的可视化学习网站。核心目标不是刷题，而是让孩子通过看、拖、调、比、分、拼等操作理解概念。

## 第一版范围

- 覆盖一年级到五年级的知识地图。
- 每个知识点都绑定一个图形化演示类型。
- 首页即为学习应用，不做营销落地页。
- 采用“数学实验室”方向：年级与主题在左侧选择，中间是大实验画布，右侧是概念解释、观察提示和操作控件。
- 使用公开目录信息整理知识点，第一版以单元级知识点为主，避免复制教材原文。

## 学生体验要求

- 低年级知识点必须使用具体物体或直观图形，例如数块、苹果片、钟表、方格纸。
- 高年级知识点必须展示结构关系，例如面积推导、分数等分、小数数轴、体积层叠。
- 页面文字要短，重点放在图形和交互反馈。
- 控件必须清楚：拖动、加减、切分、旋转、比较、播放推导。
- 每个知识点至少有一个可见的视觉演示，不能只显示文字。

## 第一版交互要求

- 年级筛选：一年级到五年级。
- 知识点列表：按年级展示，点击后切换实验。
- 实验画布：根据知识点类型渲染不同图形演示。
- 控件：每个演示类型至少支持一个参数调整。
- 学习引导：显示“看一看 / 拖一拖 / 想一想”三段式提示。
- 进度感：显示当前年级主题数量与当前知识点位置。

## 非目标

- 不做账号、登录、后端、数据库。
- 不复制教材全文或图片。
- 不做完整题库。
- 不做六年级内容。
- 不依赖远程服务才能运行。

## 验收标准

- 本地打开 `index.html` 可直接使用。
- 1-5 年级都有知识点。
- 每个知识点数据都有可视化类型，且界面可渲染对应演示。
- 至少包含数字、运算、时间、角、面积、分数、小数、统计、体积等不同演示。
- 自动测试脚本通过。
