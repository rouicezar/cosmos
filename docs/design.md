# 小学数学可视化站点设计

## 产品结构

第一版采用单页应用结构：

- 顶部：站点名称、当前实验摘要、搜索框。
- 左侧：年级选择与学习进度。
- 中间：实验画布，是页面视觉重心。
- 右侧：当前知识点、三步引导、交互控件。
- 底部：知识点切换与同年级主题进度。

## 视觉方向

方向为“数学实验室”：

- 像明亮教室里的实验台，不像后台系统。
- 主色使用白色、湖蓝、草绿、番茄红、暖黄，避免单一色系。
- 大面积留白配合清晰边界，保证小学生能快速看懂。
- 图形元素使用 CSS 和 DOM 实现，保持轻量、可交互、可测试。

## 知识点模型

每个知识点包含：

- `id`
- `grade`
- `title`
- `unit`
- `domain`
- `visual`
- `summary`
- `prompt`

`visual` 决定渲染器，例如：

- `number-blocks`
- `ten-frame`
- `number-line`
- `groups`
- `clock`
- `ruler`
- `angle`
- `fraction`
- `decimal`
- `area-grid`
- `shape-sort`
- `bar-chart`
- `volume-cubes`
- `balance`
- `optimization`

## 演示设计原则

- 数的认识：用数量块、十格框、数轴，帮助孩子建立数量感。
- 运算：用合并、拿走、分组、阵列，展示加减乘除的含义。
- 时间与测量：用钟表、尺子、单位刻度，展示量感。
- 图形与几何：用角度扇形、方格纸、可分类图形，展示空间关系。
- 分数小数：用等分条、圆片、数轴位置，展示“整体的一部分”。
- 统计：用条形图，展示数据比较。
- 体积：用层叠小方块，展示长宽高共同决定体积。

## 响应式策略

- 桌面：三栏布局。
- 窄屏：顶部、年级、实验、说明纵向堆叠。
- 实验画布保持固定最小高度，避免切换知识点时页面跳动严重。

## 可访问性

- 按钮有明确文本。
- 当前选中项使用 `aria-pressed`。
- 控件使用原生 `input[type=range]`。
- 动效克制，不依赖动效表达唯一含义。
